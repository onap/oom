1. main components included in umbrella chart
-- api-handler-infra
-- asdc-controller
-- bpmn-infra
-- openstack-adapter
-- sdnc-adapter
-- vfc-adapter 
2. dependencies included in charts sub directory
-- db-secrets
-- ssl-secrets
-- catalog-db-adapter
-- request-db-adapter
3. use global.environment to set environment to deploy to.  Possible values are dev, e2e-s1, e2e-s2, e2e-ti, ist, ist-ti, prod-s1, prod-s2, psl
4. use global.secretdeploy to enable/disable deployment of secrets.  Possible values are true and false
5. use global.secretenvironment to set environment to deploy secrets.  Possible values are dev, e2e, e2e-ti, ist, ist-ti, prod
6. use global.mso-{component name}.configmap, global.mso-{component name}.deployment, global.mso-{component name}.service flags to enable/disable which object(s) in each mso component(s) to deploy
7. Run the following helm command to deploy chart(s), remove --dry-run option
	helm upgrade -f ./helm/values-dev.yaml --install --debug --dry-run so ./so
8. there are nine (9) variation of values.yaml files each representing a different environment with environment specific values
9. replace ./helm/values-dev.yaml values yaml file to install in to different environments 	
	9.1 for example: 
		helm upgrade -f ./helm/values-ist.yaml --install --debug --dry-run --tiller-namespace=com-att-ist helm ./helm
	9.2 if there's only config changes, then you need to force the pods to restart by adding --recreate-pods --force
		helm upgrade -f ./helm/values-ist.yaml --install --debug --dry-run --recreate-pods --force --tiller-namespace=com-att-ist helm ./helm
	9.3 if you prefer to export tiller namespace rather than specifying it in the option:
		export TILLER_NAMESPACE=com-att-ist
		helm upgrade -f ./helm/values-ist.yaml --install --debug --dry-run --recreate-pods --force helm ./helm
